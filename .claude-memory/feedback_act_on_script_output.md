When running idea generation for a country-specific topic, PRIORITIZE tools in the local language if an existing hub already has tools there.

**Why:** During Finland 2026 trend research, I ran build-multilang.py and saw /fi/ had 12 tools, ran autocomplete and saw 8-9 Finnish keyword suggestions, ran validation and saw ZERO cannibalization — but still defaulted to English ideas first. User had to call this out.

**How to apply:**
1. When the input topic is about a specific country, check if we have a hub in that language FIRST
2. If yes: generate local-language tool ideas BEFORE English ones
3. Don't chase English volume numbers (29K/mo) when local-language keywords have zero competition + Tier A RPM + existing hub authority
4. Running scripts is not enough — ACT on what they tell you. The data was there, I just ignored it.
5. Low competition + high RPM + hub authority > raw volume with millions of competitors
