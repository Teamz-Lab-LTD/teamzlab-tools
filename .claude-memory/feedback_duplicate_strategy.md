---
name: Duplicate Tool Strategy
description: When duplicate found — improve if it has traffic, skip if no traffic and build new keyword instead. Revenue-first approach.
type: feedback
---

When a duplicate tool is found during building:
1. **Has traffic** → Improve it (add FAQs, fix SEO, better JS, more content)
2. **No traffic** → Skip it, build a new high-RPM keyword instead

**Why:** Improving a page Google already indexed = faster ROI than ranking a new page from scratch. But wasting time on a zero-traffic page is worse than building a new high-RPM tool.

**How to apply:** Always check Search Console data or git log age before deciding. Default: improve existing > build duplicate > skip.
